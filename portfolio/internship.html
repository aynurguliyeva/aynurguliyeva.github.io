Overview

Pension administration platforms process large volumes of financial calculations each year (contributions, reserves, interest accrual, eligibility rules). Manual validation doesn’t scale, and rule-based checks tend to miss unknown failure modes.

Goal

Detect meaningful anomalies without hard-coding brittle business rules.

Core challenge

No ground-truth labels + many “weird but valid” cases → false positives must be controlled.

Approach

Start unsupervised for discovery, then evolve into hybrid ML with Human-in-the-Loop feedback.

Key responsibilities

A practical mix of data engineering, modeling, and operationalization — with a strong focus on explainability and review workflows.

  • Designed and implemented a plan-agnostic anomaly detection workflow

    Data prep → feature engineering → model training → scoring → review-ready reporting.

  • Built robust data cleaning & validation for large datasets

    Schema checks, missing values, malformed rows, reproducible outputs.

  • Evaluated and combined multiple anomaly detection methods

    Ensemble detection + threshold calibration to keep false positives operationally manageable.

  • Implemented Human-in-the-Loop feedback for continuous improvement

    Human labels override injected labels; retraining reduces recurring false positives over time.

  • Operationalised the workflow using AWS SageMaker Pipelines

    Processing → training → batch scoring → retraining, with reproducible runs and logging.

AWS automation & MLOps

The final PoC was operationalised using AWS SageMaker pipelines, enabling reproducible processing, training, scoring, and retraining cycles.

Below is an excerpt of the AWS SageMaker pipeline used during the internship. Each block represents a reproducible processing or training step with versioned inputs and outputs.

Pipeline stages

  • Ingestion from S3
  • Cleaning + feature construction
  • Training + threshold selection
  • Batch scoring to S3
  • Retraining with labels

Monitoring

  • CloudWatch logs & metrics
  • Run tracking (reproducibility)
  • Failure visibility + alerts

Governance mindset

Focus on auditability, controlled false positive volume, and human oversight as part of the decision workflow.

Human-in-the-Loop (HITL)

A key insight: “statistically unusual” is not always “wrong”. Expert feedback turns anomaly flags into a learning signal.

Review output

  • Scored CSV stored in S3
  • Experts label: true anomaly / false positive
  • Labels merged back by policy identifier

Retraining logic

  • Start with injected training set (both classes)
  • Override injected labels with human labels
  • Train supervised model + recalibrate threshold
