Problem Statement & Context

The Challenge: Manual Validation at Scale

Context

Pension Architects processes calculations for multiple pension funds (PF1-PF4), each with thousands of policies. Each policy undergoes annual calculations for:

  • Contributions and interest accrual
  • Reserve calculations
  • Eligibility rule applications
  • Regulatory compliance checks

Existing Process

Manual validation by domain experts, with rule-based systems for common issues. This approach had several limitations:

  • Scalability: Manual review doesn't scale with data volume
  • Coverage: Rule-based systems miss unknown patterns
  • Consistency: Human reviewers have varying thresholds
  • Latency: Days to weeks for complete validation cycles

The Core Problem

"How to detect meaningful anomalies in pension calculations without ground-truth labels, while controlling false positives to maintain operational efficiency?"

Key Insight: The Hybrid Approach

The solution evolved from pure unsupervised learning to a hybrid approach: unsupervised methods for initial discovery, supervised models for refinement, and human feedback for continuous improvement. This addressed the label scarcity while maintaining operational efficiency.

Technical Implementation

Unsupervised Ensemble Design

Isolation Forest

Detects anomalies by isolating observations using random partitioning. Effective for high-dimensional data with no assumptions about distribution.

Autoencoder

Neural network trained to reconstruct normal data. High reconstruction error indicates anomalies. Captures complex non-linear patterns.

PCA + DBSCAN

Dimensionality reduction followed by density-based clustering. Identifies outliers as points in low-density regions.

Threshold Strategy for Operational Efficiency

Instead of maximizing detection rate, we optimized for operational efficiency:

Target FP Rate ≤5%
Optimal Threshold Percentile: 95th
Review Workload ~5% of policies

This approach ensured sustainable operations:

  • • Manageable volume for human reviewers
  • • Consistent review quality
  • • Scalable across pension funds
  • • Predictable operational costs

Key Benefits of HITL Approach

Continuous Improvement

Each review cycle improves model accuracy and reduces recurring false positives.

Domain Knowledge Integration

Experts' understanding of "business anomalies" captured in the model.

Reduced Review Workload

Over time, fewer false positives mean less time spent on manual review.

Audit Trail

Complete record of human decisions for compliance and debugging.
